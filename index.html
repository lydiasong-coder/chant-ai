AI詠唱練習題

作業一：API 資料整理與格式化

內容：

透過 AI 從指定 API 抓取資料，將原本較難閱讀的 API 回傳內容整理成清楚、易讀的資料格式，並輸出成 Excel。 同時將原始英文欄位名稱翻譯為繁體中文，例如將 account 顯示為會員帳號。 另外，依照測試規則計算各遊戲在追殺局中玩家贏錢的比例，若比例超過 5%，則於 log 中特別標示警示訊息。 最後再將整理完成的結果自動上傳至 Google Sheet，方便後續查看與共享。

作業二：依據作業一產出 Allure 報表

內容：

根據作業一的資料整理流程，進一步設計測試案例並進行驗證，確保 API 資料抓取、欄位轉換、資料格式化與異常判斷皆符合預期結果。 本次測試包含多項驗證項目，例如 API 是否成功回傳資料、英文欄位是否正確轉為繁體中文、資料是否正確輸出為 Excel，以及玩家贏錢比例計算是否準確， 並檢查當比例超過 5% 時是否有正確輸出警示 log。 最後將所有測試執行結果整合至 Allure 報表中，透過視覺化方式呈現測試案例、執行結果與成功失敗狀態， 讓整體測試流程與結果更加清楚，方便檢視與後續分析。

作業三：使用 VSCode 部署到 GitHub

內容：

使用 VSCode 建立並整理一頁式作業頁面，將作業一與作業二的內容整合成可閱讀的 HTML 頁面。 完成頁面後，透過 Git 與 GitHub 進行版本控管與上傳，將專案部署到 GitHub，方便保存、展示與分享。

作業四：Taiga 拉單轉 Excel

內容：

透過 Taiga 專案看板，將指定欄位中的 Bug 單資料拉出來整理成 Excel 檔案。 主要針對 NEW BUG 欄位中的問題單進行資料擷取，包含 Bug 編號、標題、狀態、建立者、內容與相關欄位資訊。 透過自動化方式將看板上的 Bug 資料整理成表格，方便後續統計、追蹤與回報。
